# Almere D6 public-source boundary

## Summary
Current sprint: Sprint 25.5 - D6 stakeholder validation handoff.

This note closes the repository-side public-source review for D6 Almere. It states what public sources can prove, what they suggest, and what they cannot settle without stakeholder validation or local/internal documents. No row in the D6 responsibility register is `settled`.

Gap classes used below:

- `public-source gap`: a likely public source may still exist, but has not yet been ingested or verified.
- `stakeholder-validation gap`: the answer requires confirmation by the responsible local or regional actor.
- `internal-document gap`: the answer likely exists in a local working document, mandate, project plan or validation record that is not in the public corpus.
- `financial-decision gap`: the answer requires finance/controller validation or a funding decision.
- `governance-decision gap`: the answer requires a mandate, ownership or classification decision.

## Closeout Rule

Repository staff should not keep expanding broad public search. Only perform targeted public-source search when a row has a named public-source gap and a likely source type. All other unresolved points should be handed to policymakers as validation, internal-document, financial-decision or governance-decision gaps.
